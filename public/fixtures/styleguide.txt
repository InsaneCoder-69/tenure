Never use the TypeScript `any` type; always declare concrete types or `unknown` with runtime type guards.
All asynchronous functions must be wrapped in `try/catch` blocks with typed catch parameters to ensure robust error handling.
Only Next.js route page files should use `export default`, as all other modules must use named exports for better tree-shaking.
React component props interfaces must use the `Props` suffix instead of an `I` prefix.
Custom React hooks must start with the `use` prefix to comply with linting rules.
All `useEffect` dependency arrays must be exhaustive, and suppressing the `react-hooks/exhaustive-deps` rule is strictly prohibited.
Remove all `console` logging statements before merging code and use the `@/lib/logger` utility for structured logging instead.
